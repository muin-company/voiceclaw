# Contributing

First off, thank you for considering contributing to this project! It's people like you that make open-source great.

## How to Report Bugs

When reporting a bug, please include as much detail as possible to help us reproduce the issue:
*   **Use a clear and descriptive title** for the issue to identify the problem.
*   **Describe the exact steps** to reproduce the problem.
*   **Provide specific examples** to demonstrate the steps (e.g., code snippets, error messages).
*   **Describe the behavior** you observed after following the steps and point out what exactly is the bug.
*   **Explain the expected behavior** and why it's expected.
*   **Include your environment details** (OS, Node.js version, package manager, etc.).

## How to Suggest Features

We welcome feature requests! Please provide the following information:
*   **Use a clear and descriptive title** for the issue to identify the suggestion.
*   **Provide a step-by-step description** of the suggested enhancement in as much detail as possible.
*   **Explain why this enhancement would be useful** to most users.
*   **List some other applications or projects** where this enhancement exists, if applicable.

## Development Setup

To set up the project locally:
1.  Fork the repository and clone your fork.
2.  Install dependencies (we typically use `npm` or `pnpm` depending on the project, check the `package.json` or lockfiles).
3.  Create a new branch for your feature or bugfix (`git checkout -b feature/your-feature-name` or `bugfix/issue-number`).
4.  Run the tests or build scripts to ensure everything is working correctly before making changes.

## Code Style

*   Follow the existing code style in the repository.
*   If the project uses ESLint, Prettier, or another formatter/linter, ensure your code passes all checks before submitting a PR.
*   Write clear, concise, and descriptive commit messages.

## Pull Request Process

1.  Ensure your branch is up to date with the `main` branch.
2.  Run all tests and formatters locally.
3.  Submit a Pull Request targeting the `main` branch.
4.  Describe your changes in the PR description, linking to any relevant issues.
5.  Wait for review. We may suggest some changes or improvements.
6.  Once approved, a maintainer will merge your PR.
